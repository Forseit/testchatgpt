# Генератор автоматических тестов

Проект предоставляет графическое приложение, которое помогает быстро создавать
файл `test.py`, запускать пользовательский скрипт на подготовленных входных
данных и собирать результаты в удобной таблице. Интерфейс позволяет задать
формат входных данных, автоматически сформировать примеры и сохранить готовый
pytest-модуль.

## Быстрый старт

1. Убедитесь, что установлен Python 3.11+.
2. Запустите графическое приложение:

   ```bash
   python main.py
   ```

3. В открывшемся окне:

   * Укажите путь к скрипту, который нужно протестировать (по умолчанию это
     прилагаемый `script.py`).
   * Настройте директорию и имя файла, куда будет сохранён сгенерированный
     модуль с тестами (`test_generated.py`).
   * Отрегулируйте параметры генератора, чтобы автоматически получить набор
     входных данных. Для примера можно оставить настройки по умолчанию и
     нажать «Сгенерировать примеры» – текстовая область заполнится готовыми
     тестами.
   * При необходимости отредактируйте входные данные вручную. Каждый тест
     разделяется пустой строкой. Строки, начинающиеся с `#`, трактуются как
     комментарий и имя теста. Чтобы указать ожидаемый ответ, добавьте строку
     с `=>`, а ниже разместите предполагаемый вывод.

4. Нажмите кнопку «Создать test.py и запустить». Приложение автоматически:

   * проверит наличие `pytest` и при необходимости установит его через `pip`;
   * сгенерирует модуль с тестами в выбранной директории;
   * прогонит ваш скрипт по каждому тесту, замерит время выполнения и соберёт
     stdout/stderr;
   * при наличии ожидаемых ответов запустит `pytest` и отобразит итоговый
     отчёт.

5. Результаты отображаются в отдельном окне: таблица с краткой информацией
   по каждому тесту, подробности по выделенной строке и (опционально)
   полный вывод `pytest`.

## Формат определения тестов

```
# Тест 1
5
1
0
1
2
0
=>
3

# Тест 2
5
1 2 3 4 5
=>
15
```

* Пустая строка разделяет тесты.
* Любая строка, начинающаяся с `#`, используется как название теста и не
  передаётся в stdin.
* Строка `=>` (или `->`, `EXPECTED:` и т.п.) делит данные на вход и ожидаемый
  результат. Ожидаемый вывод сравнивается с stdout запускаемого скрипта.

## Пример тестируемого скрипта

В репозитории есть пример `script.py`, который читает количество элементов,
последовательность чисел и выводит их сумму. Вы можете заменить его на свой
скрипт — приложение работает с любым Python-файлом.

## Структура проекта

* `main.py` — графическое приложение на Tkinter.
* `script.py` — пример целевого скрипта.
* `test_runner/` — вспомогательные модули для парсинга тестов, генерации файла
  под `pytest` и запуска пользовательского скрипта.
